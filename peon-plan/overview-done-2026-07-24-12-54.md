# Model Loading Bug Fix — Append on Agent Switch

## Goal

When switching agents whose model isn't in the current list, append that model to the existing list instead of replacing it — preserving full model choice across agent switches as documented.

## Bug

`onAgentChange()` calls `actionsBar.setModel(modelName)` which **replaces** the entire `availableModels` list with a single-item list. The doc specifies `actionsBar.addAndSelectModel(modelName)` — append to the existing list.

**Impact:** Switching between agents with different models destroys the previously loaded model list. The user loses the ability to select models from other agents.

## Design Decisions

- Add `addAndSelectModel(String modelId)` to `ActionsBarWidget` — appends the model if not already present, then selects it. Uses `AiModel.builder().id(modelId).name(modelId).build()` for consistency with `setModel()`'s pattern.
- `onAgentChange()` replaces `setModel()` call with `addAndSelectModel()`.

## Affected Files

- `/org.sterl.llmpeon/src/org/sterl/llmpeon/parts/widget/ActionsBarWidget.java` — add `addAndSelectModel()` method
- `/org.sterl.llmpeon/src/org/sterl/llmpeon/parts/AIChatView.java` — change `onAgentChange()` to use `addAndSelectModel()`

## Steps

1. Add `addAndSelectModel(String modelId)` to `ActionsBarWidget`:
   ```java
   public void addAndSelectModel(String modelId) {
       if (containsModelId(modelId)) {
           selectModel(modelId);
           return;
       }
       var model = AiModel.builder().id(modelId).name(modelId).build();
       var list = new ArrayList<AiModel>(availableModels);
       list.add(model);
       availableModels = list;
       modelCombo.setItems(list.stream().map(AiModel::getName).toArray(String[]::new));
       selectModel(modelId);
   }
   ```

2. In `AIChatView.onAgentChange()`, replace `actionsBar.setModel(aiService.getActiveModel())` with `actionsBar.addAndSelectModel(aiService.getActiveModel())`.

## Verification

- Build + run. Switch between agents with different models — model list should grow, never shrink.
